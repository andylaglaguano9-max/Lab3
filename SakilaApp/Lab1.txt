RESUMEN
En esta práctica se desarrolló una aplicación web robusta orientada a la gestión de datos utilizando el framework ASP.NET Core MVC y Entity Framework (EF) Core para establecer la conexión con una base de datos PostgreSQL existente denominada "Sakila". El trabajo tuvo como propósito principal implementar el enfoque Database First, permitiendo generar de forma automática los modelos de dominio y el contexto de datos (DbContext) directamente desde el esquema relacional.
Para ello, se configuró el entorno local instalando los paquetes NuGet necesarioss, se definió la cadena de conexión en el archivo de configuración y se procedió con el scaffolding de las entidades. Posteriormente, se desarrollaron módulos CRUD funcionales para gestionar Actores y Películas, garantizando la correcta lectura y manipulación de registros. Además, se resolvieron cinco retos analíticos utilizando sentencias de Language Integrated Query (LINQ), aplicando operadores
como Where, OrderByDescending y Take para filtrar, ordenar y agrupar datos de manera eficiente. Como etapa final, se integraron mejoras visuales en la interfaz gráfica utilizando el framework CSS Bootstrap, optimizando la presentación de las tablas y los botones de acción. Las pruebas realizadas confirmaron que la aplicación es capaz de traducir adecuadamente las consultas LINQ a SQL nativo, delegando el procesamiento al motor PostgreSQL. En conclusión, la práctica permitió afianzar conocimientos sobre el patrón MVC, el mapeo objeto-relacional inverso y el uso de consultas integradas en lenguajes fuertemente tipados.
Palabras clave: ASP.NET Core, Entity Framework Core, LINQ, PostgreSQL, Database First.
INTRODUCCIÓN:
En el desarrollo contemporáneo de software empresarial, la eficiencia en la comunicación entre la lógica de negocio y las bases de datos relacionales es un factor crítico. Históricamente, los desarrolladores debían escribir de forma manual extensas sentencias SQL combinadas con código de aplicación, lo cual propiciaba vulnerabilidades como la inyección SQL y dificultaba el mantenimiento del sistema frente a cambios en la estructura de los datos (Freeman, 2022). Frente a esto, las herramientas de Mapeo Objeto-Relacional (ORM) han revolucionado la forma de persistir datos, permitiendo tratar las tablas como clases y los registros como objetos nativos del lenguaje.
Dentro del ecosistema .NET, Entity Framework Core se destaca como el ORM oficial, ofreciendo múltiples enfoques de desarrollo. Cuando se trabaja en entornos corporativos, es habitual encontrarse con bases de datos preexistentes o sistemas heredados (Legacy). En este escenario, el enfoque Database First resulta imprescindible, ya que automatiza la creación del modelo de clases (ingeniería inversa) a partir del esquema físico de la base de datos, ahorrando horas de codificación manual y reduciendo drásticamente los errores de mapeo (Smith, 2021).
Por otro lado, la extracción y análisis de la información requiere de lenguajes de consulta potentes. Language Integrated Query (LINQ) resuelve esta necesidad al integrar capacidades de consulta de manera declarativa directamente en C#. Esto permite a los programadores filtrar, ordenar y proyectar colecciones de datos utilizando una sintaxis tipada y evaluada en tiempo de compilación, la cual es posteriormente traducida a sentencias SQL altamente optimizadas para motores robustos como PostgreSQL (Obe &amp; Hsu, 2017).
Finalmente, en el presente laboratorio se desarrolló una solución práctica orientada a implementar la conexión de una aplicación web MVC a la base de datos Sakila. A través de la configuración de paquetes, el scaffolding de entidades y el uso de LINQ para la resolución de retos específicos, estas actividades permitieron aplicar conceptos actuales de ingeniería de software, arquitectura de datos y diseńo de interfaces en un entorno local, comprobando su funcionamiento de manera visual y funcional.

MARCO TEÓRICO:
Arquitectura de ASP.NET Core MVC
El patrón Modelo-Vista-Controlador (MVC) es un paradigma de arquitectura de software que separa la aplicación en tres componentes interconectados. En el contexto de ASP.NET Core, el Modelo representa la capa de acceso y estructura de los datos, la Vista se encarga de la interfaz gráfica servida al cliente (HTML/CSS), y el Controlador actúa como intermediario procesando las peticiones HTTP y orquestando la respuesta. Según Freeman (2022), esta separación de responsabilidades no solo mejora la escalabilidad del código, sino que resulta fundamental en entornos de desarrollo ágil, ya que permite que diseńadores de UI y programadores back-end trabajen de forma paralela sin afectar la lógica core del sistema.
Entity Framework Core y el paradigma "Database First"
Entity Framework (EF) Core es un framework de mapeo objeto-relacional (ORM) ligero y multiplataforma. Facilita la interacción con bases de datos al eliminar la necesidad de usar ADO.NET clásico y comandos SQL en duro. El paradigma Database First es una estrategia en la que el esquema de la base de datos dicta la estructura de las clases en el código fuente. Según Smith (2021), a través de herramientas de interfaz de línea de comandos (CLI), EF Core lee los metadatos del motor relacional y genera automáticamente tanto las clases de entidad plana (POCO) como el DbContext encargado del rastreo de cambios. Esto es esencial para integrar aplicaciones modernas con repositorios de datos legados.
Integración con PostgreSQL y consultas LINQ
PostgreSQL es reconocido a nivel mundial como uno de los sistemas de bases de datos relacionales orientados a objetos más avanzados y confiables, caracterizado por su robustez transaccional y su compatibilidad con el estándar SQL (Obe &amp; Hsu, 2017). La comunicación entre C# y PostgreSQL se logra fluidamente a través del proveedor Npgsql. Una vez establecida la conexión, los desarrolladores utilizan LINQ (Language Integrated Query) para interactuar con los datos. LINQ introduce capacidades de consulta funcionales dentro del lenguaje C#, permitiendo usar operadores como .Where(), .OrderBy() y .Select(). La gran ventaja de LINQ en el contexto de EF Core es que la consulta se construye en memoria como un árbol de expresiones y, al momento de ejecutarse, el proveedor Npgsql la traduce al dialecto SQL exacto de PostgreSQL, logrando ejecuciones rápidas y libres de inyecciones SQL (Liberty &amp; MacDonald, 2008).

public async Task<IActionResult> Index()

9. Implementación Final (Buscador, Duración y Paginación)
Esta es la versión final optimizada para el listado principal de Películas, la cual implementa el buscador combinado y divide la información en páginas de 10 registros utilizando los operadores .Skip() y .Take(), logrando un gran rendimiento y previniendo la sobrecarga de datos en red.
public async Task<IActionResult> Index(string? buscar, int? duracionMinima, int pagina = 1)
{
    int tamanioPagina = 10;
    var consulta = _context.Films.Include(f => f.Language).AsQueryable();

    if (!string.IsNullOrWhiteSpace(buscar))
    {
        consulta = consulta.Where(f => f.Title.Contains(buscar));
    }

    if (duracionMinima.HasValue)
    {
        consulta = consulta.Where(f => f.Length >= duracionMinima.Value);
    }

    int totalRegistros = await consulta.CountAsync();

    var peliculas = await consulta
        .OrderBy(f => f.Title)
        .Skip((pagina - 1) * tamanioPagina)
        .Take(tamanioPagina)
        .ToListAsync();

    ViewBag.Buscar = buscar;
    ViewBag.DuracionMinima = duracionMinima;
    ViewBag.PaginaActual = pagina;
    ViewBag.TotalPaginas = (int)Math.Ceiling(totalRegistros / (double)tamanioPagina);

    return View(peliculas);
}

DISCUSIÓN:
El desarrollo de esta práctica permitió contrastar la teoría de la arquitectura MVC con la implementación real de una solución basada en datos. En primer lugar, la aplicación del enfoque Database First evidenció una ventaja técnica significativa frente al diseńo manual de clases. Al utilizar herramientas de ingeniería inversa para generar los modelos (POCOs) a partir de la base de datos preexistente, se garantizó una fidelidad absoluta entre el esquema físico de PostgreSQL y la lógica de dominio en C#, eliminando el margen de error humano en la definición de tipos de datos y mapeo de llaves foráneas.
Durante la implementación de las operaciones CRUD y la resolución de retos analíticos, quedó expuesta la potencia del motor Entity Framework Core en la gestión del acceso a datos. La utilización de expresiones LINQ representó un cambio de paradigma respecto a las consultas SQL incrustadas como texto plano; al interactuar con el DbContext, el código se vuelve fuertemente tipado y seguro, delegando al compilador la tarea de verificar la coherencia de las propiedades. Además, se comprobó que el ORM evalúa las consultas de forma inteligente mediante el patrón de Ejecución Diferida, asegurando que el motor de PostgreSQL asuma el procesamiento pesado (como contar registros o buscar coincidencias de texto) y enviando a la capa web únicamente la información final solicitada.
Por último, la fase de mejora visual sirvió como prueba empírica del principio de separación de preocupaciones (Separation of Concerns). La facilidad con la que se inyectaron componentes de Bootstrap en las Vistas HTML, sin necesidad de alterar una sola línea de código en los Controladores o Modelos subyacentes, reafirma que el patrón

MVC permite una alta escalabilidad. Esto demuestra que la arquitectura elegida facilita el mantenimiento del software a largo plazo, posibilitando que la interfaz de usuario y la lógica de negocio evolucionen de forma independiente.
CONCLUSIONES:
La aplicación del enfoque Database First mediante Entity Framework Core demostró ser una estrategia altamente eficiente para la integración de bases de datos preexistentes. El uso de la herramienta de scaffolding automatizó la generación de las clases de dominio (Modelos) y el contexto de datos a partir del esquema de PostgreSQL, garantizando la consistencia estructural del proyecto y reduciendo significativamente los tiempos de desarrollo al evitar la configuración manual del mapeo objeto-relacional.
